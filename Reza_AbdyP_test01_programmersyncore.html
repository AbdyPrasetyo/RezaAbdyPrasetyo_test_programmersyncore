1. Mengapa setiap bahasa pemrograman itu terdapat framework, apa kegunaan framework ?

Framework pada bahasa pemrograman dibuat untuk membantu pengembang dalam membangun aplikasi dengan cara yang lebih efisien, konsisten, dan terorganisir. Kegunaan framework sendiri yaitu untuk:

  • Meningkatkan Produktivitas
  • Konsistensi Kode
  • Mendapatkan Fitur Keamanan
  • Pengurangan Biaya dan Waktu Pengembangan
  • Banyaknya Dukungan Komunitas yang dapat mempermudah dalam membangun aplikasi

2. Jelaskan mengenai fungsi dan penggunaan if else, while, while do, switch case, for, foreach?

a. IF Else

IF Else digunakan untuk membuat keputusan berdasarkan kondisi yang diberikan jika kondisi bernilai "true" maka blok kode dalam "IF" akan dikesekusi,jika salah "false" maka blok kode dalam "Else" akan dieksekusi.

Penggunaan If Else pada javascript:
let nilai = 75;
if (nilai >= 60) {
    console.log("Lulus");
} else {
    console.log("Tidak Lulus");
}

b. While

While digunakan untuk iterasi atau pengulangan suatu blok kode selama kondisi yang diberikan bernilai benar.

Penggunaan While pada javascript:
let count = 0;
while (count < 5) {
    console.log("Pengulangan " + count);
    count++;
}

c. Do While

Do While digunakan untuk iterasi atau pengulangan suatu blok kode setidaknya sekali, sebelum kondisi diperiksa. 

let count = 0;

Penggunaan Do While pada javascript:

do {
    console.log("Pengulangan " + count);
    count++;
} while (count < 5);

d. Switch Case

Switch Case digunakan untuk membandingkan satu variabel dengan beberapa nilai.

Penggunaan Switch Case pada javascript:
let bulan = 4;
switch (bulan) {
    case 1:
        console.log("Januari");
        break;
    case 2:
        console.log("Februari");
        break;
    case 3:
        console.log("Maret");
        break;
    case 4:
        console.log("April");
        break;
    default:
        console.log("Bulan tidak diketahui");
        break;
}

e. For

For digunakan untuk mengulangi suatu blok kode untuk jumlah tertentu, dengan kontrol lebih pada variabel iterasi.

Penggunaan For pada javascript:
for (let i = 0; i < 5; i++) {
    console.log("Pengulangan ke-" + i);
}

f. Foreach

Foreach digunakan untuk mengulangi elemen-elemen dalam suatu koleksi atau array dengan lebih sederhana tanpa perlu menggunakan indeks.

Penggunaan Foreach pada javascript:
let buah = ["Apel", "Jeruk", "Mangga"];
buah.forEach(function(b) {
    console.log(b);
});

3. Apa itu constructor, method, class, object, variabel, parameter, function, procedure ?

a. Constructor

Constructor adalah class yang dipanggil saat sebuah objek dibuat. Constructor biasanya digunakan untuk menginisialisasi keadaan objek, seperti memberikan nilai awal pada variabel-variabelnya.

b. Method

Method adalah fungsi yang didefinisikan di dalam class dan digunakan untuk mendefinisikan perilaku dari sebuah objek. Method biasanya dapat mengakses data yang disimpan dalam objek dan juga mengubah status dari objek tersebut.

c. Class

Class adalah definisi kode yang digunakan untuk membuat objek. Class mendefinisikan variabel dan method yang dimiliki oleh objek.

d. Object

Object adalah instance dari sebuah class. Objek memiliki identitas, state, dan perilaku.

e. Variable

Variable adalah tempat penyimpanan yang memiliki nama dan digunakan untuk menyimpan nilai atau informasi yang dapat berubah selama program dijalankan.

f. Parameter

Parameter adalah variabel yang digunakan untuk menerima nilai dari luar saat sebuah fungsi atau metode dipanggil.

f. Function

Function adalah fungsi yang dirancang untuk melakukan tugas tertentu, dapat dipanggil ketika diperlukan, dan dapat mengembalikan nilai. Function dapat memiliki parameter untuk menerima input.

g. Procedure

Procedure digunakan untuk menekankan bahwa operasi yang dilakukan tidak menghasilkan output atau return value.

4. Perbedaan method=”Post” dan “Get” ?

- GET digunakan untuk mengambil data dari server. Data yang diminta disertakan dalam URL sebagai bagian dari permintaan.
- POST digunakan untuk mengirim data baru atau terupdate ke server. Data tersebut dikirim melalui body dari permintaan dan tidak terlihat di URL.

5. Jelaskan konsep mengenai web service dan fungsi apa yang digunakan untuk memanggil web service client ?

Web service adalah sebuah layanan yang tersedia melalui web, yang memungkinkan aplikasi untuk berkomunikasi atau bertukar data antara satu sama lain, Web service menggunakan standar internet untuk mengirim dan menerima pesan data, melalui protokol HTTP. Format data yang sering digunakan dalam web service adalah XML (eXtensible Markup Language) dan JSON (JavaScript Object Notation).

- Untuk memanggil web service dari client, biasanya digunakan HTTP request methods seperti GET, POST, PUT,PATCH dan DELETE. Fungsi atau metode yang digunakan untuk melakukan pemanggilan ini tergantung pada bahasa pemrograman atau platform yang digunakan, namun konsepnya serupa di mana sebuah HTTP request dikirimkan dan respons diharapkan dari server.

6. Jelaskan mengenai MVC dan HMVC !

a. MVC (Model View Controller)

MVC adalah sebuah desain arsitektural untuk memisahkan representasi data dari interaksi pengguna.
- Model adalah Komponen yang bertanggung jawab atas data, logika bisnis, dan aturan aplikasi. Model menyimpan data dan keadaan aplikasi, biasanya berinteraksi dengan database atau layanan lain untuk mengelola data.
- View adalah komponen yang menangani representasi visual dari data, yaitu apa yang pengguna lihat di layar. View mempresentasikan data dari Model kepada pengguna.
- Controller adalah sebagai perantara antara Model dan View. Controller menerima input dari pengguna, memprosesnya (dengan mungkin mengubah Model), dan mengirimkan data kembali ke View untuk diperbarui.

b. HMVC (Hierarchical Model View Controller)

HMVC (Hierarchical Model View Controller) adalah ekstensi dari pola MVC yang dengan konsep hierarki dalam komponen MVC.
- Modul dalam HMVC, setiap modul adalah sub-sistem yang lengkap dengan Model, View, dan Controller sendiri. Modul-modul ini dapat berinteraksi tetapi umumnya dirancang untuk operasi mandiri, yang bersifat modular.
- Hierarki dalam HMVC memungkinkan aplikasi untuk disusun dalam struktur yang lebih teratur, memfasilitasi pembagian aplikasi yang besar menjadi bagian-bagian yang lebih kecil dan lebih mudah dikelola.

9. Perbedaan HTTP dan HTTPS ?

  • - HTTP tidak memiliki fitur enkripsi bawaan.Data yang dikirim dan diterima melalui HTTP dapat dengan mudah disadap atau dimodifikasi oleh pihak ketiga selama transmisi, yang dikenal sebagai serangan "man-in-the-middle".
  • - HTTPS, menggunakan protokol SSL/TLS untuk mengenkripsi data yang ditransmisikan. Ini membuat data tidak bisa dibaca oleh pihak ketiga yang mungkin mencoba mengintersepsi transmisi tersebut. Enkripsi ini melindungi terhadap penyadapan dan memastikan integritas data.

10. Jelaskan Apa itu API? sebutkan contohnya?

API (Application Programming Interface) adalah sekumpulan aturan dan definisi yang memungkinkan aplikasi perangkat lunak berkomunikasi satu sama lain.API berperan sebagai jembatan yang memungkinkan satu program untuk mengakses fitur atau data dari aplikasi lain, tanpa perlu mengetahui detail implementasi internal dari aplikasi tersebut.

contoh API:

Kegunaan = Mengintegrasikan peta, pencarian lokasi, dan fitur navigasi ke dalam aplikasi web atau mobile.
Contoh= penggunaan sebuah aplikasi pengiriman makanan menggunakan Google Maps API untuk menampilkan peta kepada pengguna dan membantu pengemudi menemukan alamat pengiriman dengan tepat.
